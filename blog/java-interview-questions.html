Java Interview Questions

50 concise Q&A to refresh core Java knowledge before your interview.

← Back to Blog

  1. 1) JDK vs JRE vs JVM?

    JVM runs bytecode; JRE = JVM + libraries to run Java apps; JDK = JRE + tools (javac, javadoc) to develop and run.

  2. 2) What are the four OOP pillars?

    Encapsulation, Inheritance, Polymorphism, Abstraction.

  3. 3) Why is String immutable?

    Safety for class loaders/security, caching in string pool, thread-safety, and stable hash codes for use as keys.

  4. 4) StringBuilder vs StringBuffer?

    Both are mutable; StringBuffer is synchronized (thread-safe, slower). StringBuilder is non-synchronized (faster, single-threaded).

  5. 5) equals() and hashCode() contract?

    Equal objects must have equal hashCodes. If you override equals, override hashCode to maintain collections consistency.

  6. 6) List vs Set vs Map?

    List: ordered, duplicates. Set: unique elements. Map: key→value pairs, keys unique.

  7. 7) ArrayList vs LinkedList?

    ArrayList: fast random access, slower inserts/removes in middle. LinkedList: fast inserts/removes at ends, poor random access.

  8. 8) How does HashMap work?

    Computes hash of key, finds bucket, resolves collisions via linked list/tree bins; equals() confirms key matching.

  9. 9) ConcurrentHashMap vs HashMap?

    ConcurrentHashMap supports safe concurrent access with internal segmentation; HashMap is not thread-safe.

  10. 10) Fail-fast vs fail-safe iterators?

    Fail-fast throw ConcurrentModificationException; fail-safe iterate over a snapshot and allow concurrent mods.

  11. 11) final, finally, finalize?

    final: non-changeable; finally: block that always runs; finalize(): deprecated GC callback, don’t use.

  12. 12) Checked vs unchecked exceptions?

    Checked must be declared/handled (IOException); unchecked extend RuntimeException (NullPointerException).

  13. 13) try-with-resources?

    Auto-closes resources implementing AutoCloseable, reducing leaks and boilerplate.

  14. 14) Functional interface?

    An interface with a single abstract method (SAM). Enables lambdas; examples: Runnable, Callable, Comparator.

  15. 15) Lambda vs anonymous class?

    Lambdas are concise for functional interfaces; anonymous classes can implement multiple methods or capture more context.

  16. 16) Stream intermediate vs terminal ops?

    Intermediate return a Stream (lazy): map, filter. Terminal produce a result/side effect: collect, forEach, reduce.

  17. 17) When to use Optional?

    To model optional returns, not fields. Avoid using it in collections or as method parameters.

  18. 18) record classes?

    Immutable data carriers with auto-generated equals/hashCode/toString and canonical constructor.

  19. 19) var and type inference?

    Local variable type inference; improves readability when the type is obvious from the initializer.

  20. 20) Switch expressions (Java 14+)?

    Use arrow labels and yield values; avoids fall-through, more concise than traditional switch.

  21. 21) Sealed classes (Java 17)?

    Restrict which classes can extend/implement a type, enabling exhaustive handling and stronger modelling.

  22. 22) ClassLoader basics?

    Loads classes into the JVM; follows parent delegation (bootstrap → platform → application) for security and consistency.

  23. 23) JVM memory areas?

    Heap (objects), Stack (frames), Metaspace (class metadata), Code cache, and Native memory.

  24. 24) Strong vs soft vs weak vs phantom refs?

    GC priority: strong retained; soft cleared under memory pressure; weak cleared eagerly; phantom used for cleanup tracking.

  25. 25) What is garbage collection?

    Automatic reclamation of unreachable objects; collectors include G1, ZGC, Shenandoah (low pause goals).

  26. 26) volatile keyword?

    Ensures visibility of writes across threads and prevents reordering; does not provide atomicity for compound actions.

  27. 27) synchronized vs ReentrantLock?

    synchronized is simpler (intrinsic lock). ReentrantLock offers tryLock, fairness, interruptible waits, multiple conditions.

  28. 28) wait/notify/notifyAll usage?

    Call on the monitor within synchronized blocks; wait releases the lock and suspends until notify/notifyAll signals.

  29. 29) Thread states?

    NEW, RUNNABLE, BLOCKED, WAITING, TIMED_WAITING, TERMINATED.

  30. 30) Executor framework benefits?

    Decouples task submission from execution; thread pools improve throughput and resource control.

  31. 31) Future vs CompletableFuture?

    CompletableFuture supports chaining, composition, and async callbacks; Future is a simple placeholder requiring blocking get().

  32. 32) What is a deadlock?

    Threads wait on each other’s locks cyclically; avoid by consistent lock ordering or using timeouts/tryLock.

  33. 33) Livelock vs starvation?

    Livelock: threads keep responding to each other without progress. Starvation: a thread never gets CPU/resources.

  34. 34) Comparable vs Comparator?

    Comparable defines natural ordering within the class; Comparator defines external custom orderings.

  35. 35) Serialization and transient?

    Serialization converts objects to byte streams; transient fields are skipped to protect sensitive/derived data.

  36. 36) Reflection basics?

    Inspect/modify classes at runtime (Class, Method, Field). Use carefully—can break encapsulation and hurt performance.

  37. 37) What are annotations?

    Metadata for code processed by tooling or at runtime (e.g., @Override, @Deprecated, custom annotations with retention/targets).

  38. 38) Generics and type erasure?

    Generics provide compile-time type safety; erasure removes type info at runtime, limiting reflection on generic types.

  39. 39) PECS rule for wildcards?

    Producer Extends, Consumer Super: use ? extends T to read; ? super T to write.

  40. 40) Autoboxing pitfalls?

    Nulls can cause NPE; boxed equality uses object identity, not value; performance overhead vs primitives.

  41. 41) Why BigDecimal for money?

    Avoids binary floating-point rounding errors; supports exact decimal arithmetic and scale control.

  42. 42) java.time vs java.util.Date?

    java.time is immutable, thread-safe, and comprehensive; Date/Calendar are mutable and error-prone.

  43. 43) Locale and timezone concerns?

    Always specify ZoneId/Locale when parsing/formatting; store instants in UTC, present in user’s zone.

  44. 44) Defensive copying?

    Return copies of mutable fields and accept copies in constructors to preserve encapsulation.

  45. 45) Immutable vs unmodifiable collections?

    Immutable: contents can’t change. Unmodifiable: wrapper prevents changes via that reference but underlying may mutate.

  46. 46) When to use EnumMap/EnumSet?

    When keys/elements are enums. They’re memory-efficient and fast due to array/bit-vector implementations.

  47. 47) What is a memory leak in Java?

    Unintended object retention via references (caches, listeners) preventing GC; fix by clearing references and using weak refs if apt.

  48. 48) equals() vs ==?

    == compares references for objects and values for primitives; equals() compares logical equality (if overridden).

  49. 49) Static vs instance members?

    Static belong to the class (one per class); instance belong to each object. Static methods can’t access instance state directly.

  50. 50) Best practices for equals/hashCode/toString?

    Follow contracts, include significant fields, keep them consistent with immutability; consider records for value types.
